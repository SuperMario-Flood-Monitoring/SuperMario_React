# UI 정책

작성 기준: 2026-06-29

## 기본 방향

이 프로젝트는 도시침수 관련 현장 대응 도구를 목표로 한다. UI는 장식보다 빠른 판단, 반복 사용, 정보 밀도, 조작 안정성을 우선한다.

- 주요 상태와 수치는 즉시 읽히게 표시한다.
- 작업 화면은 과한 마케팅형 구성보다 기능 중심으로 구성한다.
- 카드나 패널은 실제 정보를 묶는 용도로만 사용한다.
- 모바일은 터치 조작과 좁은 화면에서의 가독성을 우선한다.

## web/mobile 정책

web과 mobile은 같은 기능을 제공하되 화면 구조는 다를 수 있다.

- web: 그리드, 테이블, 패널 중심
- mobile: 세로 흐름, 하단 시트, 큰 터치 영역 중심
- 공통 API와 표시 로직은 services/shared에서 재사용한다.

## 테마 정책

작업장은 라이트/다크 테마를 지원한다.

- 초기 테마는 시스템 설정을 따른다.
- web 작업장에서는 테마 토글 버튼을 제공한다.
- 다크 테마에서는 테이블, 모달, 입력 요소까지 배경과 경계 색을 함께 조정한다.
- 현재 테마 선택은 브라우저 저장소에 유지하지 않는다.

## 위험 로그 UI 정책

위험 로그는 현장 조치 흐름을 방해하지 않도록 안정적인 목록 렌더링을 우선한다.

- 실시간 이벤트는 즉시 row로 추가하지 않는다.
- 새 로그 개수만 표시하고, 사용자가 새로고침을 눌렀을 때 목록을 갱신한다.
- 기본 필터는 `조치 전`, `조치 중`이다.
- `조치 후`는 기본 목록에서 제외하고 필터로 조회한다.
- 상태 필터는 `상태` 헤더 버튼에서 제공한다.
- 시간 정렬은 `시간` 헤더 버튼에서 제공한다.
- 상태 필터 버튼에는 필터 가능함을 알 수 있는 아이콘을 함께 둔다.

## 문구 정책

현장 사용자가 보는 화면에서는 내부 구현 용어를 피한다.

- 피해야 할 표현: 백엔드, 도메인, 컬럼
- 권장 표현: 서버, 업무 기준, 항목, 내용, 상태

도움말은 사용자가 당장 판단해야 하는 항목의 의미를 설명하는 데 집중한다.

## 수치 표시 정책

위험 로그 내용은 서버 원문을 그대로 노출하기보다 사용자가 이해할 수 있는 형태로 다듬는다.

- 중복 대상명은 제거한다.
- `flowCms`는 `m³/s` 단위로 표시한다.
- 긴 대상명은 필요한 경우 줄바꿈으로 보여준다.
- 위험 유형은 `REVERSE_FLOW` 같은 코드보다 `역류` 같은 한국어 표현을 우선한다.

## 모달 정책

모달 또는 하단 시트는 사용자가 선택한 row의 다음 행동만 보여준다.

- `조치 전`: 조치 내용 입력
- `조치 중`: 결과 및 재발 시 참고사항 입력
- `조치 후`: 기본 목록에서는 숨김

조치 내용을 제출한 직후 다음 단계 모달을 자동으로 열지 않는다. 제출이 성공하면 선택 상태를 해제하고, 사용자가 다시 row를 클릭했을 때 다음 입력을 시작한다.
