# 실시간 통신 처리

작성 기준: 2026-06-29

## WebSocket URL 생성

SWMM WebSocket URL은 `src/services/swmm/client.ts`의 `getSwmmWebSocketUrl`에서 만든다.

- HTTP base URL이면 `ws:`로 변환한다.
- HTTPS base URL이면 `wss:`로 변환한다.
- path는 `/ws/simulation`을 사용한다.

## 시뮬레이션 실시간 처리

시뮬레이션 화면은 SWMM 엔진이 보내는 snapshot을 받아 화면 상태를 갱신한다.

- 노드 상태
- 링크 상태
- 실행 시간
- 위험 이벤트
- 요약 정보

React는 수리 계산을 직접 수행하지 않는다. SWMM 엔진에서 계산된 값을 받아 표시한다.

## 위험 로그 실시간 처리

위험 로그 화면도 같은 시뮬레이션 WebSocket에서 위험 이벤트를 감지한다. 다만 이벤트를 즉시 row로 렌더링하지 않는다.

이 정책을 두는 이유는 사용자가 특정 row를 클릭하려는 순간 새 row가 추가되어 클릭 대상 위치가 바뀌는 문제를 줄이기 위해서다.

처리 방식은 다음과 같다.

1. WebSocket 메시지에서 위험 이벤트를 감지한다.
2. 이미 본 이벤트인지 확인한다.
3. 새 이벤트면 `새 로그` 개수를 증가시킨다.
4. 화면 목록에는 즉시 추가하지 않는다.
5. 사용자가 새로고침 버튼을 누르면 `GET /api/hazards`를 호출한다.
6. 서버 목록을 기준으로 화면을 다시 렌더링한다.

## 중복 이벤트 방지

위험 로그 화면은 이미 감지한 WebSocket 이벤트 id를 별도로 기억한다. 같은 이벤트가 여러 번 들어와도 새 로그 개수가 중복 증가하지 않도록 하기 위한 처리다.

## 연결 실패 처리

WebSocket 연결 실패나 중단은 화면 기능 전체를 막지 않는다. 위험 로그 목록은 `GET /api/hazards`로 다시 조회할 수 있고, 시뮬레이션 화면은 API 상태 조회와 엔진 제어 응답을 함께 사용한다.

실시간 연결 상태가 불안정한 경우에는 사용자가 새로고침 또는 엔진 상태 갱신을 통해 서버의 최신 상태를 다시 가져오는 방식으로 복구한다.
