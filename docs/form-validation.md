# 폼/입력값 검증

작성 기준: 2026-06-29

## 로그인 폼

로그인 폼은 아이디와 비밀번호 입력을 받는다.

- 빈 값은 제출하지 않도록 처리한다.
- 로그인 실패 시 사용자에게 오류 메시지를 표시한다.
- 로그인 성공 시 세션을 저장하고 `/simulation`으로 이동한다.

실제 인증 요청은 `src/services/auth/authApi.ts`에서 처리한다.

## 위험 로그 조치 시작 폼

`조치 전` 상태의 위험 로그를 클릭하면 조치 내용을 입력한다.

- 필수 입력값: 조치 내용
- 입력값은 제출 전 trim 처리한다.
- 빈 문자열이면 API 요청을 보내지 않는다.
- 제출 중에는 중복 제출을 막는다.

제출 payload는 다음과 같다.

```json
{
  "action_detail": "조치 내용",
  "action_type": "FIELD_CHECK"
}
```

성공하면 row 상태를 `조치 중`으로 바꾸고, 내용에는 입력한 조치 내용을 표시한다.

## 위험 로그 조치 완료 폼

`조치 중` 상태의 위험 로그를 클릭하면 결과와 재발 시 참고사항을 입력한다.

- 필수 입력값: 결과
- 선택 입력값: 재발 시 참고사항
- 입력값은 제출 전 trim 처리한다.
- 결과가 빈 문자열이면 API 요청을 보내지 않는다.
- 제출 중에는 중복 제출을 막는다.

제출 payload는 다음과 같다.

```json
{
  "result_detail": "결과",
  "result_status": "RESOLVED",
  "recurrence_note": "재발 시 참고사항"
}
```

완료 요청에는 action id가 필요하다. 화면은 상세 조회 응답의 `actions` 배열에서 진행 중인 action을 찾아 사용한다.

## 편집기 입력

편집기는 배수 객체와 연결 정보를 JSON 구조로 관리한다. 입력값은 화면 state와 `EditorLayout` 타입을 기준으로 유지한다.

편집 결과는 시뮬레이션 엔진 실행 payload로 사용되므로, 객체 id와 연결 관계가 깨지지 않도록 저장 전 구조를 유지해야 한다.

## 시뮬레이션 제어 입력

시뮬레이션 화면은 강우량, 실행 속도, 막힘 설정 등의 제어값을 입력받는다.

- 숫자 입력은 화면에서 허용 범위에 맞게 제한한다.
- 엔진 제어 요청은 `src/services/swmm/client.ts`를 통해 보낸다.
- 실행 중 변경 가능한 제어값은 `/engine/control`로 전달한다.

## 공통 제출 정책

폼 제출 중에는 같은 요청이 반복해서 나가지 않도록 제출 상태를 둔다. API 오류가 발생하면 입력값을 임의로 버리지 않고, 사용자가 다시 확인 후 제출할 수 있게 한다.
